TsetlinLab

An interactive, in-browser lab for interpretable Tsetlin Machines. View every rule in plain language, trace any sample end-to-end (inputs → bits → clauses fired → class vote → decision), and compare binarisers and engines side by side. Inference runs fully client-side in WebAssembly — no server, no install.

The binariser is decisive on hard datasets: holding the engine fixed, GLADE bits lift TON_IoT from 83.0% → 96.6% and MedSec from 87.4% → 96.8% — with fewer bits. Click any cell's view rules to open that exact model and inspect its clauses, encoding and per-sample decision paths. The last two columns isolate the contradiction question. FPTM (perara, base) is github.com/perara/Tsetlin.jl with its guard off (self-contradicting rules present). GLADE + FPTM (same-bit fix) is Tsetlin_contradiction_fix.jl, which removes every same-bit contradiction. Two kinds of contradiction exist: same-bit (one bit asked ON and OFF) and thermometer (a feature asked ≥ a high rung and < a lower rung — different bits). A same-bit fix clears the first; on a thermometer binariser (GLADE) the second remains (shown in orange). With a 1-bit-per-feature encoding (e.g. 1-bit MNIST) there are no rungs, so the same-bit fix reaches a true 0 — verified. See code/contradiction_fix/CONTRADICTION_RESULTS.md.

Takeaway: same-bit fixes (yours and perara's guard) are equivalent and reach a true 0 on 1-bit encodings; on thermometer binarisers (GLADE / Standard) they clear same-bit but leave thermometer contradictions, which need feature-group handling. The contradiction class depends on the encoding, not the engine.
